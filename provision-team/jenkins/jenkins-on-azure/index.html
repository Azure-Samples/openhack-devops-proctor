Jenkins On Azure

This Jenkins instance does not support https, so logging in through a public IP address has been disabled (it would expose your password and other information to eavesdropping). To securely login, you need to connect to the Jenkins instance using SSH port forwarding. You can find the password (jenkinsVMPassword) in the zip file provided to your team.

For Linux or Mac

ssh -L 127.0.0.1:8080:localhost:8080 jenkins@{domain-name}

For Windows

putty.exe -ssh -L 8080:localhost:8080 jenkins@{domain-name}

1- After you have started your tunnel, navigate to http://localhost:8080/ on your local machine.

2- Unlock the Jenkins dashboard for the first time with the initial admin password. To get this token, SSH into the VM and run:

sudo cat /var/lib/jenkins/secrets/initialAdminPassword
